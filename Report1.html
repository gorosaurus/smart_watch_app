Description of App

With such a small screen, reading on a smartwatch the conventional way could be difficult. By printing one word to the screen at a time in quick succession, our application would implement Rapid Serial Visual Representation, an alternative method of reading that would be ideal for the small screen of a smartwatch. RSVR implementations such as Spritz and Spreeder have been created before, but, to our group’s knowledge, one hasn’t been created for the smartwatch. Our application would access text files from the smartwatch’s file system and present them to the user with easily customisable speeds and fonts. What makes our application good for our group is that it is easily scalable, significantly decreasing our risk of failure. It could start out by interpreting .txt files before moving on to pdf, docx, and epub files along with other ebook formats. Ideally, we hope to eventually give our app HTML-reading capabilities, opening up the possibility of creating a primitive browser for the smartwatch.

Version Control System

We chose to use GitHub over other means of version control for a number of reasons. GitHub offers all of the distributed revision control and source code management functionality of Git, as well as adding it's own features.
It provides the ability to revert documents to previous revisions, providing an important system of back-ups and the ability to fork projects.
Forking can be done in one-click, which makes it easy to work on another version of the project at no risk of overwriting what is already in place. GitHub is also a free service, which means that no extra funds had to be spent on the project, and accessibility is high.

The Editor

The team has chosen to use NetBeans, which is an open source code editor. We decided to use NetBeans as, first of all, it is able to run on both windows and OSX, which were necessary for our group. NetBeans also provides functionality such as syntax highlighting, code analysis and code completion, which is very easy to use, and features drag-and-drop functionality for quickly creating GUI's, which was an extra bonus. NetBeans also features database support, including built-in functionality for SQL and Oracle. This editor also has a built in debugger, which allows you to inspect your code step-by-step to see where any issues are occurring. NetBeans also has additional support for JavaScript, the language we will be coding in, with it's integrated JavaScript editor. Finally, NetBeans also has direct support for working with GitHub, making it much easier for the team to push/pull the project when needed. When any important change is made, we will be able to ‘push’ the project to the repository and instantly have a working backup.

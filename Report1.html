Description of App

With such a small screen, reading on a smartwatch the conventional way could be difficult. By printing one word to the screen at a time in quick succession, our application would implement Rapid Serial Visual Representation, an alternative method of reading that would be ideal for the small screen of a smartwatch. RSVR implementations such as Spritz and Spreeder have been created before, but, to our group’s knowledge, one hasn’t been created for the smartwatch. Our application would access text files from the smartwatch’s file system and present them to the user with easily customisable speeds and fonts.

What makes our application good for our group is that it is easily scalable, significantly decreasing our risk of failure. It could start out by interpreting .txt files before moving on to pdf, docx, and epub files along with other ebook formats. Ideally, we hope to eventually give our app HTML-reading capabilities, opening up the possibility of creating a primitive browser for the smartwatch.

Programming Language

Javascript is an interpreted language, meaning that it is a programming language that most of its implementations execute instructions directly without previously compiling the program into machine code language. It is a dynamic programming language that execute many common programming behaviours that static programming languages perform during compilation at run time . Javascript is an untyped language. That means that it allows any operation to be performed on any data, which are generally considered to be sequences of bits of various lengths. When saying JavaScript is dynamically typed; it means that a type is associated with each value, rather than just with each expression. For example, a variable that is at one time bound to a number may later be re-bound to a string. JavaScript supports various ways to test the type of an object. Javascript can perform both as a procedural and an object oriented language. Objects are created in javascript by attaching method and properties to other empty objects at run time, and once the object is constructed it can be used as prototype for creating similar objects. It is an object oriented language with first-class functions, which means that it treats functions as first class citizen. It supports passing functions as arguments to other functions, returning them as values from other functions and assigning them to variables or storing them in data structures. It is a prototype- based scripting language. It builds up object systems by inheritances and aggregation. Inheritance is done by copying existing objects ( prototypes). According to Douglas Crockford ( an American senior JavaScript architect at PayPal, and is also a writer and speaker on JavaScript, JSON, and related web technologies) inheritance of prototypes in JavaScript is described as “: you make prototype objects, and then … make new instances. Objects are mutable in JavaScript, so we can augment the new instances, giving them new fields and methods. These can then act as prototypes for even newer objects. We don't need classes to make lots of similar objects… Objects inherit from objects. What could be more object oriented than that?” Javascripts works within the web browser, it cannot handle aggregating information from users and communicating them with other machines. It does not contact the server, thus cannot store information on the server. JavaScript does not perform server side validation, only clients side validation. Javascript works differently in different browsers.

The Editor

The team has chosen to use NetBeans, which is an open source code editor. We decided to use NetBeans as, first of all, it is able to run on both windows and OSX, which were necessary for our group. NetBeans also provides functionality such as syntax highlighting, code analysis and code completion, which is very easy to use, and features drag-and-drop functionality for quickly creating GUI's, which was an extra bonus. Finally, NetBeans also has direct support for working with GitHub, making it much easier for the team to push/pull the project when needed. When any important change is made, we will be able to ‘push’ the project to the repository and instantly have a working backup.

The Debugger

Ultimately, we have decided to use the built-in debugger which comes with NetBeans. There were many positives to using this debugger, including that we would not have to copy or paste code between two different programs. We considered using a debugger in our browser, but instead decided that everything can just be done within NetBeans, which significantly reduces complexity. It also features the use of breakpoints, allowing a user to ‘step through’ their program line-by-line.

Furthermore, NetBeans also contains a visual debugger, which allows for easier debugging on the GUI elements. We will be able to take a GUI snapshot, and then locate any problem code from a visual standpoint. This will be significantly quicker than searching through the code itself, as working with GUI code can be challenging.
